# Chat

The **in-app chat** lets you message colleagues directly inside OpenTMS — handy when you want to ask a planner about a file, ping a dispatcher about a delay, or send a quick "thanks" to a team-mate without switching to another app.

:::info Filling this page in later
A detailed walkthrough with screenshots will be added in a later phase. For now this page describes the feature at a high level.
:::

## Where to find it

The **chat icon** in the [app bar](./app-bar) opens the chat panel. A red badge shows the number of unread messages — across all conversations.

## What you can do

| Action | How |
|---|---|
| **Start a conversation** | Click the chat icon → click **New conversation** → search for a colleague by name. |
| **Send a message** | Type in the box at the bottom of the conversation and press <kbd>Enter</kbd>. |
| **See conversation history** | Past conversations stay in the list — click one to reopen it. |
| **Get notified of new messages** | The bell badge updates in real time, and you can also enable email or browser notifications in [your settings](./user-menu#change-password). |

## Who can you chat with?

You can chat with any other user inside the same tenant. Cross-tenant chat is not supported — if you need to talk to someone outside your organization, use email or another channel.

## Privacy

Chat messages are stored on the OpenTMS server inside your tenant. Administrators with the right permissions can audit chat history if your organization's policy requires it. Do not use chat for content that should not be retained — use the appropriate channel instead.
