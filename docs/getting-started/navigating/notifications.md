# Notifications

OpenTMS uses **notifications** to keep you informed about events that need your attention without forcing you to refresh screens or hunt through lists. Things like a new transport file assigned to you, a status change on a file you're following, or a long-running export that just finished.

## Where to find them

The **bell icon** in the [app bar](./app-bar) shows the number of unread notifications. Click it to open the notifications panel.

The bell is only visible if your account has the **Notifications** permission. Ask your administrator if it's missing.

## What you can do from the panel

| Action | What it does |
|---|---|
| **Read** | Click a notification to jump to the related screen (for example, the transport file it refers to). |
| **Mark as read** | Single notification or all of them at once. Read notifications are dimmed in the list. |
| **Settings** | Open notification preferences to choose which events you want to be notified about. |

## Notification settings

You can fine-tune which events trigger a notification for you, and how:

1. Click the bell, then click the **settings** icon at the top of the panel — or open the [user menu](./user-menu) → **My settings → Notification preferences**.
2. For each event type (e.g. *Transport file assigned to me*, *Status change*, *Export finished*) you can choose:
   - **In-app** — show in the bell.
   - **Email** — send an email as well.
   - **Off** — never notify me about this.
3. Click **Save**.

:::tip
If you find yourself drowning in notifications, turn off the ones you don't act on and keep only the ones that genuinely require a response.
:::

## Real-time delivery

Notifications appear in real time — no need to refresh the page. As soon as the event happens on the server, the bell updates and shows the new count.
